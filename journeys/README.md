# Journeys (use cases)

A **journey** is *what* to accomplish — a concrete use case with steps and success criteria.
A **persona** (see `../personas/`) is *who* is doing it and *how* they behave.

At invocation you pick one of each:

> Run the **confused-user** persona through the **add-asset-with-reminder** journey.

The journey supplies the goal and the "done" bar; the persona supplies the mindset, pace, and
quirks. If no journey is given, the skill falls back to the persona's own default goal.

## Authoring a new journey

Copy `_template.md`, fill it in, drop it here. Keep it about user intent, NOT implementation —
describe what the user is trying to do and how you'd know they succeeded, not which buttons to
click (the persona discovers those live, honest-eyes style).
